### 当前交互流程
```
打开 App → 空白画布
  ├─ 双击画布 → 搜索添加节点
  ├─ 左侧图标 → 打开节点库拖拽添加
  ├─ 右键 → 上下文菜单
  ├─ 连线 → 手动拖拽输入→输出
  ├─ 选中节点 → 属性面板配置参数
  ├─ 顶部 ▶ → 运行工作流
  └─ 悬停节点 → 预览输出数据
```

### 优点
- ComfyUI 式浮层面板，最大化画布空间
- 搜索优先的节点添加方式（效率高）
- 多工作流标签切换

### 待改进
- **没有明确的"入口"**：打开即空白画布，用户不知道该干什么
- **节点库扁平化**：74 个节点全部挤在分类折叠树里，找起来累
- **数据流不可见**：节点间数据传递在"黑盒"里，执行完才知道结果
- **没有执行反馈**：运行时节点上无状态指示
- **缺少"操作主线"**：用户不知道"标准流程"是什么

## 二、推荐的操作逻辑

### 核心设计原则

| # | 原则 | 为什么 |
|---|------|--------|
| 1 | **Dashboard 入口** | 用户需要看到自己的工作流库，这是"家" |
| 2 | **模板驱动** | 新手从模板开始，不用从零搭建 |
| 3 | **数据流可见** | 每一步的数据变化在画布上直接可见 |
| 4 | **节点即状态机** | 每个节点有明确的状态：待执行→执行中→成功→失败 |
| 5 | **快捷键优先** | 高频操作用键盘，降低鼠标依赖 |

### 推荐的完整操作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    1️⃣ Dashboard（入口）                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 🆕 新建     │  │ 📋 从模板   │  │ 📂 打开...  │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│  工作流网格（卡片列表）                                       │
│  ┌────┐ ┌────┐ ┌────┐                                       │
│  │竞品 │ │日报 │ │数据 │  ...                               │
│  │监控 │ │生成 │ │清洗 │                                     │
│  └────┘ └────┘ └────┘                                       │
└───────────────────────┬─────────────────────────────────────┘
                        │ 点击"新建"或选择模板
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                2️⃣ Canvas（编辑器）                            │
│                                                             │
│  🏠 侧边栏  ┌──────────────────────────┐  📋 属性面板        │
│  (折叠)    │                          │  (选中节点时)       │
│  ┌──┐     │   🌐 HTTP 请求            │                    │
│  │🔍│     │   URL: https://api...     │  节点: HTTP 请求    │
│  └──┘     │   GET ▼   ┌──┐─○         │  URL: https://...   │
│  ┌──┐     │           │连线│          │  方法: GET          │
│  │📦│     │   ┌───────┘   │          │                    │
│  └──┘     │   ▼           ▼          │  📊 输出预览:       │
│  ┌──┐     │  🔀 JSON    📊 保存      │  ┌──────────────┐  │
│  │📊│     │  解析       CSV          │  │ { data: [...] │  │
│  └──┘     │                          │  └──────────────┘  │
│  ┌──┐     │                          │                    │
│  │⚙│      │      ← 无限画布 →        │                    │
│  └──┘     │                          │                    │
│  ┌──┐     └──────────────────────────┘                    │
│  │🏠│                                                     │
│  └──┘     ┌──────────────────────────────────────────────┐│
│           │  📋 执行日志                                  ││
│           │  [10:30:01] HTTP 请求 → 200 (234ms)           ││
│           │  [10:30:02] JSON 解析 → 42 条记录             ││
│           │  [10:30:02] 保存 CSV → output.csv (8KB)       ││
│           └──────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 交互细节

#### 2.1 节点添加（3种方式）
| 方式 | 操作 | 场景 |
|------|------|------|
| 双击搜索 | 画布空白双击 → 输入关键词 → Enter 添加 | **主力方式**，最快 |
| 节点库拖拽 | 侧栏打开 NodePalette → 展开分类 → 拖到画布 | 浏览型，探索可用节点 |
| 右键菜单 | 右键 → "添加节点" → 搜索 | 备选 |

> **推荐**：把双击搜索作为默认引导。空画布上显示大号提示 "双击搜索节点"。

#### 2.2 节点连线
- 按住输出针脚 → 拖到目标输入针脚 → 松手连线
- 类型匹配验证：不同类型针脚之间禁止连线（颜色区分，拒绝时红色闪烁）
- **自动连线建议**：拖拽节点靠近已有节点时，自动吸附并对齐

#### 2.3 节点配置
- 选中节点 → 右侧属性面板弹出（当前已实现）
- 属性面板内容由节点类型决定，用小部件（widget）渲染：
  - `string` → 文本输入框
  - `number` → 数字输入 + 步进
  - `combo` → 下拉选择
  - `toggle` → 开关
  - `text` → 多行文本

#### 2.4 执行（4种模式）
| 模式 | 快捷键 | 行为 |
|------|--------|------|
| **全量运行** | `Ctrl+Enter` | 从入口节点开始，按 DAG 拓扑顺序全量执行 |
| **单步执行** | `Ctrl+.` | 只执行下一个待执行的节点 |
| **从选中运行** | 右键 → "从此运行" | 断点续跑 |
| **定时执行** | 顶栏 → 定时计划 | cron 表达式，后台调度 |

#### 2.5 运行时可视化
- **节点状态色环**：
  - ⚪ 白色 = 待执行（idle）
  - 🔵 蓝色闪烁 = 执行中（running）
  - 🟢 绿色 = 成功（success）
  - 🔴 红色 = 失败（error）+ 错误信息悬停可见
- **数据流动画**：执行完成时，连线上短暂显示数据流动粒子效果
- **输出预览**：鼠标悬停在有输出的节点右侧 → 工具提示显示 JSON 摘要

#### 2.6 调试
- **断点**：右键节点 → "设置断点"，执行到该节点时暂停
- **数据查看**：点击已执行的节点 → 属性面板展示完整输出
- **变量面板**：debug 模式下显示当前上下文所有变量

## 三、Dashboard vs Canvas 入口策略

### 推荐：Dashboard 为默认入口

理由：
1. 这是"桌面应用"而非"在线工具"。用户打开应用是为了处理某个任务
2. 工作流文件管理是刚需（已有 workflow_list/list_templates）
3. 从 Dashboard 到 Canvas 是自然过渡（新建/打开/从模板）

### Dashboard 布局建议
```
┌─────────────────────────────────────────────────────┐
│  🔍 搜索工作流...                 🆕 新建  📋 从模板  │
├─────────────────────────────────────────────────────┤
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐│
│  │ 📊 竞品价格   │ │ 📧 日报生成   │ │ 🔄 数据清洗   ││
│  │  监控        │ │              │ │              ││
│  │ 上次: 2h前   │ │ 上次: 昨天   │ │ 上次: 5天前  ││
│  │ ✅ 12 节点   │ │ ⚠ 6 节点    │ │ ✅ 8 节点    ││
│  └──────────────┘ └──────────────┘ └──────────────┘│
│  ┌──────────────┐ ...                              │
│  │ ...          │                                  │
│  └──────────────┘                                  │
├─────────────────────────────────────────────────────┤
│  ⚡ 模板库                                         │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐│
│  │ 🛒 网页数据   │ │ 📊 Excel    │ │ 📄 文档处理   ││
│  │  抓取        │ │  报表生成    │ │  批量替换     ││
│  └──────────────┘ └──────────────┘ └──────────────┘│
└─────────────────────────────────────────────────────┘
```

### 过渡动画
- Dashboard → Canvas：卡片展开动画，画布从卡片中央放大
- Canvas → Dashboard：按 Esc 或点击侧栏 Home 按钮，画布缩小回卡片

## 五、"无代码"体验闭环

对于新手用户，完整闭环：

```
1. 打开应用 → 看到自己的 Dashboard
2. 点击 "从模板" → 选择 "网页数据抓取" 模板
3. 画布打开，节点已经铺好（HTTP → JSON解析 → 保存CSV）
4. 用户只需：双击 HTTP 节点 → 填写 URL → 点运行
5. 看到节点依次变绿，数据流过连线
6. 保存 CSV 节点变绿 → 悬停看到保存的文件路径
7. 点 Dashboard 打开输出文件
```

对于进阶用户：
```
1. 新建空白工作流
2. 双击搜索 "HTTP" → 添加
3. 双击搜索 "JSON" → 添加
4. 连线 HTTP.result → JSON.data
5. 双击搜索 "CSV" → 添加为输出
6. 配置参数 → 运行 → 完成
```
