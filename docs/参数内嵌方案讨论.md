# 参数内嵌方案讨论

## 当前方案：右侧面板
```
步骤列表                    参数面板
┌──────────────────┐      ┌──────────────┐
│ 🌐 浏览器操作      │      │ 导航到页面    │
│   ● 导航到页面    │ ←点击 │ URL: [____]  │
│   ● 点击登录      │ ───→ │ 等待: [load] │
│   ● 输入用户名    │      │ 🔗 引用变量  │
└──────────────────┘      └──────────────┘
```
问题：需要来回点击，变量名和参数分离

## 内嵌方案：参数直接放动作框里
```
┌────────────────────────────────┐
│ 🌐 浏览器操作                    │
│                                │
│ ● 导航到页面            导航   │
│   URL: [https://___]           │
│   等待: [load ▾]               │
│   🔗 引用: [— 选择变量 —]      │
│                                │
│ ● 点击登录按钮      点击_点击  │
│   选择器: [#login] 🎯          │
│   🔗 引用: [— 选择变量 —]      │
│                                │
│       ＋ 增加动作               │
└────────────────────────────────┘
```

## 优点
- 所见即所得，不用切换视角
- 变量名 + 参数 + 引用 在同一上下文
- 少一次点击

## 需要考虑
- 卡片会变长 → 需要展开/收起机制
- 建议：默认收起只显示变量名，点击展开编辑

## 交互方案
1. **收起态**：只显示动作名 + 变量名（当前已实现）
2. **展开态**：点击动作框 → 展开参数编辑区（内嵌，不是右侧面板）
3. 再点一次 → 收起

是否按这个方向做？
