# v4.0 容器化重构计划

> 伟哥指令："简化节点，以容器为载体，只保留 Browser/Excel/Word + IF 分叉逻辑节点"

## 核心变化

| 维度 | 现在 (v3.x) | 目标 (v4.0) |
|------|-----------|------------|
| 节点类型 | 20+ 原子操作 | 4 种容器 + IF |
| 节点内部 | 单 action | 多 action 顺序执行 |
| 连线含义 | 数据流 = 执行序 | 数据流（端口→端口） |
| 数据传递 | widget + `{{}}` 引用 | 端口命名 + 连线对接 |

## 容器定义

**Browser 容器** — 一个浏览器窗口内完成所有操作：
```
输入口: 填入值, 搜索关键词, ...
输出口: 提取文本, 页面标题, 截图, ...
内部动作: 导航/等待/输入/点击/提取/截图/滚动/执行JS...
```

**Excel 容器** — 一个 Excel 文件内完成所有操作：
```
输入口: 写入位置, 写入值, ...
输出口: 单元格值, 范围数据, 行数, ...
内部动作: 读单元格/写单元格/筛选/排序/公式...
```

**Word 容器** — 一个 Word 文件内完成所有操作：
```
输入口: 替换值, 插入内容, ...
输出口: 段落文本, 表格数据, ...
内部动作: 读段落/替换文本/插入表格/合并...
```

**IF 容器** — 条件分叉：
```
输入口: 条件值
输出口: true出口, false出口
内部动作: 条件判断
```
