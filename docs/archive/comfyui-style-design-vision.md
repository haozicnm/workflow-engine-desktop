## 一、核心理念

**工作流的本质是数据流。** 每一个节点 = 一个数据加工站，用户不需要写代码，只需要：

1. 从节点库拖一个节点到画布
2. 填写参数（像填表单）
3. 连线（数据从上个节点流向下个节点）
4. 点运行，看结果

### 设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **零代码体验** | 80% 场景用纯配置节点覆盖，代码节点是最后选项 |
| 2 | **数据可见** | 每个节点的输入/输出数据在画布上直接可见或悬停预览 |
| 3 | **类型安全** | 针脚按类型颜色编码，类型不匹配插不上 → 天然防呆 |
| 4 | **紧凑高效** | 节点参数直接暴露在节点上，不隐藏在面板里 |
| 5 | **即时反馈** | 运行状态实时显示在节点上，数据流过连线时有动画 |

### 三大区域

| 区域 | 位置 | 功能 |
|------|------|------|
| **节点库** | 左侧面板 | 按分类折叠排列，支持搜索过滤，拖到画布即添加 |
| **画布** | 中央 | 无限画布，节点拖拽/连线/删除，缩放平移 |
| **属性面板** | 右侧面板 | 选中节点时显示参数配置 + 输入输出数据预览 |

### 底部

- **执行日志**：运行时的实时日志输出
- **控制台**：可手动输入表达式/命令调试

### 工具栏功能

| 按钮 | 功能 |
|------|------|
| ▶ 运行 | 执行整个工作流 |
| ⏯ 单步 | 逐个节点手动触发（调试模式） |
| ■ 停止 | 终止当前执行 |
| ⚙ 设置 | 工作流级别的配置（超时、重试等） |
| ↻ | 自动布局（整理节点排列） |
| 📁 | 工作流列表 / 新建 / 保存 |

## 三、节点设计

### 3.1 节点结构

```
┌─────────────────────────────────────┐
│                                     │
│  🌐 HTTP 请求            ● 运行中   │  ← 标题栏（类型图标 + 名称 + 状态灯）
│  ─────────────────────────────────  │
│  📥 输入                            │
│  ├─○ headers  : {对象}    ← 蓝     │  ← 输入针脚（左侧）
│  └─○ body     : {对象}             │
│                                     │
│  ⚙ 参数                             │
│  URL  [https://api.example.com  ]   │  ← 可直接编辑
│  方法 [GET          ▼]              │
│  超时 [30] 秒                        │
│                                     │
│  📤 输出                            │
│  └─○ result   : {JSON}    ← 橙     │  ← 输出针脚（右侧）
│                                     │
└─────────────────────────────────────┘
```

### 3.2 针脚类型系统 + 颜色编码

| 类型 | 颜色 | 图标 | 示例 |
|------|------|------|------|
| **文本** (String) | 🔵 蓝 `#4A9EFF` | `Aa` | 一段文字、URL |
| **数字** (Number) | 🟢 绿 `#4ADE80` | `#` | 计数、金额、百分比 |
| **布尔** (Boolean) | 🟡 黄 `#FBBF24` | `◉` | true / false |
| **对象** (Object/JSON) | 🟠 橙 `#FB923C` | `{}` | API 返回的 JSON |
| **数组** (Array) | 🟣 紫 `#C084FC` | `[]` | 列表数据 |
| **文件** (File) | ⚪ 灰 `#9CA3AF` | `📄` | 文件路径或内容 |
| **任意** (Any) | ⬜ 白 `#E5E7EB` | `*` | 可以是任意类型 |
| **错误** (Error) | 🔴 红 `#EF4444` | `!` | 异常输出 |

### 3.3 状态指示灯

```
●━━ 灰色 (#9CA3AF)  →  未执行（Idle）
●━━ 黄色 (#FBBF24)  →  等待中（Queued）
●━━ 蓝色脉冲        →  执行中（Running）
●━━ 绿色 (#4ADE80)  →  成功（Success）
●━━ 红色 (#EF4444)  →  失败（Error），节点展开显示错误信息
●━━ 橙色 (#FB923C)  →  警告（Warning），非致命问题
```

### 3.4 数据预览

**方式 A：悬停预览气泡**
```
鼠标悬停在针脚上 →
┌──────────────────────┐
│ 📤 result : {JSON}   │
├──────────────────────┤
│ {                    │
│   "name": "商品A",   │
│   "price": 99,       │
│   "stock": 1523      │
│ }                    │
│ ─────────────────── │
│ 大小: 48 bytes       │
└──────────────────────┘
```

**方式 B：属性面板**  
选中节点时，右侧面板显示所有输入/输出的完整数据（适合大段数据）

### 3.5 连线样式

```
正常连线:
○────────────────────○   实线，灰色 #64748B

数据流过时:
○════════════════════○   加粗 + 绿色流动动画

悬停时:
○────────────────────○   高亮亮色
```

## 四、核心节点类型（第一版）

### 📥 数据源节点

| 节点 | 功能 | 核心参数 | 输出 |
|------|------|----------|------|
| **HTTP 请求** | 发送 HTTP 请求 | URL, Method, Headers, Body, 超时 | 响应体 (JSON/文本) |
| **网页抓取** 🕷 | 可视化抓取网页 | URL → 预览页面 → 点选元素 → 自动填充选择器 | 抓取结果数组 |
| **读取文件** | 读本地文件 | 文件路径, 编码, 格式 | 文件内容 |
| **数据库查询** | 执行 SQL 查询 | 连接配置, SQL 语句 | 查询结果 |
| **手动输入** | 手动输入常量 | 文本/数字/JSON | 输入的常量 |
| **剪贴板读取** | 读系统剪贴板 | (无) | 剪贴板内容 |

### 🔧 处理节点

| 节点 | 功能 | 核心参数 | 输出 |
|------|------|----------|------|
| **文本拼接** | 拼接/模板替换 | 模板: `你好 {姓名}，分数 {分数}` | 拼接后文本 |
| **正则提取** | 正则匹配/替换 | 正则表达式, 捕获组名 | 提取结果 |
| **JSON 解析** | 解析/提取字段 | JSONPath 表达式 | 提取的字段 |
| **JSON 构造** | 多个字段组装成 JSON | 键值对映射 | JSON 对象 |
| **数组操作** | 过滤/排序/去重/分页 | 操作类型, 条件 | 处理后的数组 |
| **数值计算** | 加减乘除/统计 | 表达式 | 计算结果 |
| **条件判断** | IF / ELSE | 条件表达式 | 分两路输出 (true/false) |
| **分支路由** | Switch 多路 | 匹配规则 | 多路输出 |
| **循环遍历** | For Each | 对数组每项执行子节点 | 结果数组 |
| **类型转换** | 转文本/数字/JSON | 目标类型 | 转换后数据 |
| **数据合并** | 合并多个输入 | 合并策略 | 合并后数据 |
| **数据拆分** | 按字段拆分 | 拆分子段 | 拆分后数据 |

### 🤖 AI 节点

| 节点 | 功能 | 核心参数 | 输出 |
|------|------|----------|------|
| **调用大模型** | 调用 LLM | Prompt, 模型, 温度 | AI 响应 |
| **文本翻译** | 翻译文本 | 源语言, 目标语言 | 翻译结果 |
| **文本总结** | 长文本摘要 | 最大字数 | 摘要 |
| **文本分类** | 分类文本 | 分类标签 | 分类结果 |
| **情感分析** | 分析情感倾向 | (无) | 正面/负面/中性 |
| **实体提取** | 提取人名/地名等 | 实体类型 | 实体列表 |

### 📤 输出节点

| 节点 | 功能 | 核心参数 | 输出 |
|------|------|----------|------|
| **保存文件** | 写入本地文件 | 路径, 格式, 编码 | (无，副作用) |
| **发送邮件** | 发送邮件 | 收件人, 主题, 正文 | (无) |
| **写入数据库** | 插入/更新数据 | 表名, 数据 | (无) |
| **导出 Markdown** | 渲染模板 → MD | 模板, 数据 | MD 文件 |
| **导出 Excel** | 生成表格 | 列定义, 数据 | XLSX 文件 |
| **写入剪贴板** | 写系统剪贴板 | (无) | (无) |
| **控制台打印** | 打印调试信息 | 文本 | (无) |

## 五、网页抓取节点（重点交互）

这是最有差异化的节点，设计如下：

```
┌──────────────────────────────────────────┐
│  🕷 网页抓取                              │
│  ────────────────────────────────────────│
│                                          │
│  URL [https://example.com/products    ]  │
│      [🔍 预览页面]                        │
│                                          │
│  ┌──── 页面预览 ──────────────────────┐  │
│  │                                    │  │
│  │  [商品列表页面]                     │  │
│  │  ┌──────────────────┐              │  │
│  │  │ 商品A    ¥99      │ ← 点击这里   │  │
│  │  │ 商品B    ¥199     │              │  │
│  │  │ 商品C    ¥299     │              │  │
│  │  └──────────────────┘              │  │
│  │                                    │  │
│  └────────────────────────────────────┘  │
│                                          │
│  提取规则:                                │
│  ├ 容器选择器  [.product-item         ]  │
│  ├ 名称字段    [.product-name         ]  │
│  ├ 价格字段    [.product-price        ]  │
│  └ [+ 添加字段]                          │
│                                          │
│  📤─○ items : [{名称, 价格}...]  ← 🟣  │
│                                          │
└──────────────────────────────────────────┘
```

### 交互流程

1. 用户填入目标 URL
2. 点击「预览页面」，节点内展开内嵌浏览器
3. 用户在页面预览中**直接点击**要抓取的元素
4. 系统自动分析元素，填充 CSS 选择器
5. 用户可继续点击更多元素，定义字段映射
6. 预览抓取结果，确认后保存

## 六、执行模式

### 6.1 运行模式

```
▶ 全量运行      一键执行所有节点，从头到尾
⏯ 单步调试      逐个节点手动触发，看到每一步的输出再决定下一步
```

单步调试时每个节点上出现一个小「▶」按钮，点一下就执行这个节点。

### 6.2 执行动画

```
    运行中的工作流:
    
    ┌──────────┐         ┌──────────┐         ┌──────────┐
    │ 🌐 HTTP  │ ═══════▶│ 🔀 JSON  │ ═══════▶│ 📊 保存  │
    │  ✅ 200  │  数据流  │  ⏳ ...  │         │  ● 等待  │
    │  230ms   │  动  画  │          │         │          │
    └──────────┘         └──────────┘         └──────────┘
```

- 连线上的绿色虚线在数据传递时流动
- 已完成节点右上角显示状态 + 耗时
- 当前执行中的节点边框有呼吸动画

### 6.3 错误处理

错误节点自动展开：
```
┌──────────────────────────────────┐
│  🌐 HTTP 请求          ● 失败    │
│  ────────────────────────────────│
│  ⚠ 连接超时                      │
│  URL: https://api.example.com    │
│  耗时: 30123ms (超过30秒限制)     │
│                                  │
│  [🔄 重试] [✕ 忽略继续]          │
└──────────────────────────────────┘
```

## 八、技术上怎么实现

### 8.1 前端（Vue 3）

```
基础库:   Vue Flow (vueflow.dev)
          ├── 节点拖拽 / 连线 / 画布缩放
          ├── 自定义节点渲染
          └── 事件系统（nodeClick, connect, etc.）

自定义:
├── 节点样式（ComfyUI 风格 CSS）
├── 针脚类型系统（颜色 + 验证）
├── 属性面板（参数编辑器）
├── 数据预览（气泡 + 面板）
├── 执行动画
└── 节点库面板（搜索 + 拖入）
```

### 8.2 后端（Rust / Tauri）

```
执行引擎:
├── DAG 拓扑排序（决定执行顺序）
├── 每个节点类型对应一个执行函数
├── 数据在节点间传递（类型检查）
├── 并行执行支持（无依赖的节点可同时跑）
└── 执行状态实时推送到前端（Tauri events）

节点注册:
├── 每个节点 = impl Node trait
├── 定义输入针脚、输出针脚、参数
└── execute(inputs, params) -> outputs
```

### 8.3 通信方式

```
前端 (Vue 3)                    后端 (Rust)
─────────────                   ────────────
拖拽节点到画布    ──────────▶    (无通信，纯前端)
配置参数          ──────────▶    (无通信，纯前端)
点击运行          ─ invoke ──▶   解析 DAG → 拓扑排序
                                    ↓
                                执行节点 1  ─ event ──▶   更新节点状态
                                    ↓
                                执行节点 2  ─ event ──▶   更新节点状态 + 数据
                                    ↓
                                执行完成    ─ event ──▶   全部完成
```
