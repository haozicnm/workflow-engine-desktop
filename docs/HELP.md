## 快速开始

1. **新建工作流**：点击左侧栏底部「＋ 新建」按钮
2. **添加步骤**：点击编辑区顶部「＋ 添加步骤」，选择容器类型
3. **配置动作**：在步骤卡片中点击「＋ 增加动作」添加具体操作
4. **运行**：点击右上角 ▶ 按钮执行工作流
5. **查看结果**：运行后各步骤输出会折叠显示在卡片底部

## 创建工作流

### 步骤类型

| 类型 | 图标 | 用途 |
|------|------|------|
| 🌐 浏览器 | 打开网页、点击、填表、截图、提取数据 |
| 📊 Excel | 读写 Excel 文件，支持追加、更新、筛选 |
| 📝 Word | 读写 Word 文档，模板替换 |
| 🔀 条件判断 | 根据数据决定后续执行路径 |
| 🌐 HTTP | 发送 HTTP 请求（GET/POST/PUT/DELETE） |
| ⏰ 延时 | 等待指定时间后继续 |
| 🔔 通知 | 发送系统通知 |
| 📜 脚本 | 执行 JavaScript/Rhai 脚本 |
| 📋 剪贴板 | 读写系统剪贴板 |
| 🔄 游标迭代 | 逐条处理数据（支持跨次执行） |
| 🔁 批量循环 | 一次性遍历全部数据 |
| ✋ 人工审批 | 暂停等待人工审核 |

### 动作

每个容器类型下有多个**动作**可选。例如浏览器容器包含：

- **打开网页**：导航到指定 URL
- **点击元素**：点击页面上的按钮/链接
- **填写表单**：在输入框中填入文本
- **提取文本**：获取元素的文本内容
- **截图**：对页面或元素截图
- **等待元素**：等待某个元素出现
- 等等…

## 变量引用

### 基本格式

变量引用使用 `{{变量名}}` 语法，在参数输入框中输入 `{{` 会自动弹出变量选择器。

```
步骤级引用：{{step_1}}          → 整个步骤的输出对象
动作级引用：{{step_1.action_1_1}} → 单个动作的输出值
```

### ID 命名规则

- 步骤 ID：`step_1`, `step_2`, `step_3`（按顺序自动编号）
- 动作 ID：`action_1_1`, `action_1_2`（`action_{步骤号}_{动作序号}`）

### 使用示例

```
步骤1：浏览器 → 打开网页 https://example.com
步骤2：浏览器 → 提取文本 .price → 输出到 step_1.action_1_1
步骤3：Excel  → 写入数据 → 引用 {{step_1.action_1_1}} 作为数据源
```

### 变量选择器

在任意参数输入框中：

1. 点击输入框右侧的 🔗 按钮，或直接输入 `{{`
2. 弹出变量选择器，按步骤分组显示
3. 点击变量名即插入引用
4. 「⚡ 整个输出」引用步骤的完整输出对象

## 条件执行

### 逻辑判断节点

1. 添加「🔀 条件判断」步骤
2. 在条件构建器中设置条件（支持 AND/OR 组合）
3. 输出 `{branch: "true"/"false", value: ..., result: true/false}`

### 条件执行（runCondition）

下游步骤可以**监听逻辑节点的分支结果**：

1. 点击目标步骤的 ⋯ 菜单 → 「🔀 条件执行」
2. 选择引用的逻辑步骤和触发条件：
   - **满足时执行**：逻辑节点输出 `branch=true` 时执行
   - **不满足时执行**：逻辑节点输出 `branch=false` 时执行
   - **总是执行**：无论真假都执行

这实现了**去中心化条件路由**——多个步骤可以分别监听同一个逻辑节点的不同分支。

## 迭代节点

### 游标迭代（🔄 游标迭代）

- **逐条处理**：每次运行只处理一项数据
- **跨次持久化**：运行结束后游标位置保存到文件，下次运行从断点继续
- **自动完成**：所有数据处理完毕后 `done=true`，可触发后续通知节点

配置：
```json
{
  "items": "{{step_1.action_1_1}}",  // 数据源（数组）
  "body": [...]                       // 子步骤
}
```

输出：`{done: false, item: 当前项, index: 0, total: 100, remaining: 99}`

### 批量循环（🔁 批量循环）

- **一次性遍历**：在单次运行中处理全部数据
- **内存模式**：适合小数据集（< 10000 条）
- **收集结果**：所有子步骤的输出收集到 `results` 数组

## 人工审批

### 基本用法

在工作流中添加「✋ 人工审批」步骤：

```
步骤1：浏览器 → 提取数据
步骤2：人工审批 → 确认数据无误后继续
步骤3：Excel → 写入数据
```

运行到审批步骤时会**暂停等待**，右下角出现「✋ 待审批」浮动按钮。

### 自定义选项

```json
{
  "title": "价格审核",
  "message": "订单金额：{{step_1.action_1_1.金额}} 元",
  "options": "同意,拒绝,需要更多信息",
  "recommended": "同意",
  "timeout": 300,
  "timeout_action": "recommended"
}
```

- **推荐选项**：UI 上高亮显示（⭐），引导快速决策
- **自动决策**：设置 `require_review: false` 可跳过审批，直接用推荐选项
- **超时策略**：
  - `recommended`：超时后自动执行推荐选项
  - `reject` / `approve`：超时后强制拒绝/通过
  - `fail`：超时后标记工作流失败

### 审批面板

- 右下角浮动按钮显示待审批数量
- 点击展开侧边面板，显示所有待审批列表
- 支持超时倒计时、审批意见输入、查看上游数据

## Excel 操作

### 支持的动作

| 动作 | 说明 | 示例 |
|------|------|------|
| 📖 读取 | 读取工作表数据 | `{sheet: "Sheet1", range: "A1:D10"}` |
| ✏️ 写入 | 写入数据到指定位置 | `{sheet: "Sheet1", data: [[...]], start_cell: "A1"}` |
| ➕ 追加 | 在已有数据后追加行 | `{sheet: "Sheet1", data: [["新行"]]}` |
| 🔧 更新 | 原地更新指定单元格 | `{sheet: "Sheet1", updates: [{cell: "A2", value: "新值"}]}` |
| 📋 创建 | 创建新的 Excel 文件 | `{sheets: ["Sheet1", "Sheet2"]}` |
| 🔍 筛选 | 按条件筛选行 | `{column: "状态", value: "已完成"}` |
| 📊 排序 | 按列排序 | `{column: "金额", order: "desc"}` |
| 📑 工作表 | 列出所有工作表名 | `{}` |
| 📤 列提取 | 提取指定列数据 | `{column: "A"}` |
| 🔄 CSV转换 | Excel ↔ CSV 互转 | `{direction: "xlsx_to_csv"}` |

### 原地写入（格式保留）

`追加` 和 `更新` 动作使用 umya-spreadsheet 库，**原地修改已有文件**，完整保留：
- ✅ 字体、颜色、边框
- ✅ 列宽、行高
- ✅ 公式
- ✅ 合并单元格

## 浏览器自动化

### 支持的动作（31 个）

**基础操作**：打开网页、点击、填写、提取文本/HTML、截图、滚动、等待

**高级操作**：执行 JavaScript、获取属性、悬停、拖拽、键盘输入、选择下拉框

**智能等待**：等待元素出现/消失/可点击

**验证**：检查元素是否存在、文本是否匹配

### 元素选择器

支持 CSS 选择器和文本选择器：

```
#submit          → ID 选择器
.btn-primary     → 类选择器
button[type=submit]  → 属性选择器
text=登录        → 文本选择器（精确匹配）
```

## 定时执行

1. 打开工作流 → 点击 ⋯ 菜单 → 「⏰ 定时」
2. 设置 cron 表达式（如 `0 9 * * *` = 每天 9:00）
3. 启用后自动按计划执行

常用 cron 表达式：
- `0 9 * * 1-5` — 工作日 9:00
- `*/30 * * * *` — 每 30 分钟
- `0 0 1 * *` — 每月 1 号

## 调试技巧

### 断点调试

1. 点击步骤卡片左侧的 ● 圆点设置断点
2. 运行工作流，执行到断点时自动暂停
3. 查看当前变量值和步骤输出
4. 点击 ▶ 继续执行，或单步执行

### 单步执行

点击 ⋯ 菜单 → 「单步执行」，每执行一个步骤后暂停。

- 当前执行步骤

- 总耗时

## 常见问题

### Q: 变量引用没有生效？

**检查清单**：
1. 引用格式是否正确：`{{step_1.action_1_1}}`（不是 `{{step_1.action_1_1.output}}`）
2. 被引用的步骤是否在当前步骤之前
3. 步骤 ID 是否匹配（看步骤卡片标题旁的 ID）

### Q: 浏览器操作超时？

- 默认超时 30 秒，可在「等待元素」动作中调整
- 检查选择器是否正确
- 网页加载慢时，先用「等待元素」等待关键元素出现

### Q: Excel 写入后格式丢失？

- 使用「追加」或「更新」动作（原地修改，保留格式）
- 「写入」动作会重建文件，格式丢失

### Q: 工作流运行失败怎么排查？

1. 查看状态栏的错误信息
2. 检查失败步骤的输出（展开卡片查看）
3. 使用断点调试逐步排查
4. 检查变量引用是否正确

### Q: 如何导入/导出工作流？

- **导出**：点击工作流 → ⋯ 菜单 → 导出为 JSON 文件
- **导入**：将 JSON 文件拖拽到左侧栏，或点击「导入」按钮

### Q: 审批超时后会怎样？

根据 `timeout_action` 配置：
- `recommended`：自动执行推荐选项
- `reject`：自动拒绝
- `approve`：自动通过
- `fail`：工作流标记为失败
