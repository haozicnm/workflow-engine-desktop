# 工作流执行逻辑全面修复方案

## 问题总览

| # | 问题 | 严重度 | 影响 |
|---|------|--------|------|
| 1 | 变量引用格式完全错误 | 🔴 致命 | 所有变量引用都不工作 |
| 2 | action.id ≠ action.label 导致数据断流 | 🔴 致命 | 容器内数据无法正确流转 |
| 3 | ActionPanel 冗余未清理 | 🟡 中等 | UI 混乱 |
| 4 | 步骤输出格式无统一约定 | 🟡 中等 | 下游引用不一致 |

---

## 问题 1：变量引用格式完全错误

### 现状
- **前端生成**：`{{浏览器操作_点击登录.output}}`（用 step.label + action.label 拼接）
- **后端期望**：`{{stepId.actionLabel}}`（step.id 做根 key，action.label 做子 key）

### 后端解析链路
```
{{browser1.提取文本}}
  → root = "browser1" → step_outputs["browser1"]
  → .get("提取文本") → 输出值
```

### 修复方案
**引用格式改为**：`{{stepId.actionLabel}}`

**前端改动**：
- `ActionRow.vue` 的 `varName` computed：改为用 `step.id` + `action.label`
- `ActionRow.vue` 的 `availableRefs`：用 `step.id` 做引用 ID，`step.label` 做显示名
- `ActionPanel.vue`（清理前同步修复）
- `LogicBranch.vue` 的变量 picker

**显示 vs 存储分离**：
- 显示：`浏览器操作 › 提取文本`（友好）
- 存储：`{{browser1.提取文本}}`（机器可读）

## 问题 2：action.id ≠ action.label 导致数据断流

### 现状
- **输入端口**：按 `action.id` 查找 → `input_ports.get("action_1_in")`
- **输出端口**：按 `action.label` 写入 → `output_ports.insert("提取文本", data)`

如果 `action.id = "act_abc123"` 而 `action.label = "提取文本"`，输入输出完全对不上。

### 修复方案
统一用 `action.label` 做 key（因为 label 是用户可读的，也是变量引用的目标）。

**后端改动**：
- `browser_container.rs`：输入端口改为按 `action.label` 查找
- `excel_container.rs`：同上
- `word_container.rs`：同上

---

## 问题 3：ActionPanel 冗余

### 现状
- `Editor.vue` 仍然 import 并渲染 `ActionPanel`
- 点击动作行还会触发 `onActionClick` 打开右侧面板
- 参数编辑已内嵌到 `ActionRow`，右侧面板完全多余

### 修复方案
- 删除 `Editor.vue` 中 ActionPanel 相关代码（import、template、state、handlers）
- 删除 `selectedActionId`、`selectedAction`、`onActionClick`、`onUpdateActionParams`、`onClosePanel`
- 保留 `ActionPanel.vue` 文件（不删文件，只清引用）

---

## 问题 4：步骤输出格式统一

### 现状
- 数据容器（browser/excel/word）：输出 `{actionLabel: value}` 对象
- 逻辑容器：输出 `{branch, value, result}` 固定结构
- 变量引用统一用 `.output` 但实际没有 `.output` 这一层

### 修复方案
**输出格式约定**：

```json
// 数据容器（browser/excel/word）
{
  "提取文本": "hello",
  "点击登录": { "success": true }
}

// 逻辑容器
{
  "branch": "true",
  "value": "原始值",
  "result": true
}
```

**变量引用规则**：
- 引用整个步骤输出：`{{stepId}}` → 完整 JSON 对象
- 引用特定动作输出：`{{stepId.动作label}}` → 单个值
- 引用逻辑分支结果：`{{stepId.branch}}` / `{{stepId.result}}`

**前端变量选择器显示**：
- 步骤级：`🌐 浏览器操作` → 引用 `{{browser1}}`
- 动作级：`🌐 浏览器操作 › 提取文本` → 引用 `{{browser1.提取文本}}`

---

## 修复顺序

1. **问题 2**（action.id/label 统一）— 后端基础，不改这个后面都白搭
2. **问题 1**（变量引用格式）— 前端核心，让引用真正能工作
3. **问题 3**（清理 ActionPanel）— UI 清理
4. **问题 4**（输出格式文档化）— 确认并固化
