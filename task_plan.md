### v2.0 — ComfyUI 风格编辑器改造 🚀

> 设计文档：[docs/comfyui-style-design-vision.md](docs/comfyui-style-design-vision.md)
> 核心理念：工作流 = 数据流。节点 = 数据加工站，零代码拖拽连线。
> 技术栈：Vue Flow（vueflow.dev）+ Rust/Tauri（DAG 执行引擎）

| Phase | 任务 | 状态 |
|-------|------|------|
| **P1: 基础节点编辑器** | Vue Flow 画布、ComfyUI风格节点、连线+针脚类型系统、节点库面板、基础节点（HTTP/JSON/文本/文件） | ⬚ 未开始 |
| **P2: DAG 执行引擎** | Rust 端 DAG 解析、节点执行框架、前后端实时通信（Tauri events）、执行动画+状态显示 | ⬚ 未开始 |
| **P3: 高级节点** | 网页抓取（内嵌浏览器+点选）、AI 节点（LLM/翻译/分类）、条件判断/循环、数据预览完善 | ⬚ 未开始 |
| **P4: 增强** | 子图/模块化、工作流模板市场、导入/导出、性能优化 | ⬚ 未开始 |

**设计决策（待你确认）：**
| # | 决策 | 选择 |
|---|------|------|
| 1 | 执行模式 | 全量运行 + 单步调试 |
| 2 | 数据预览 | 悬停气泡 |
| 3 | 子图/模块化 | 第一版就做 |
| 4 | 代码节点 | 保留一个兜底 |
| 5 | 并行执行 | 默认串行，手动标记并行 |

## 技术债务

| 项 | 说明 | 优先级 |
|----|------|--------|
| RHAI thread_local | tokio work-stealing 下可能跨线程，建议 spawn_blocking 包装 | 低 |
| Step O(n) 查找 | `scheduler.rs` 中每次循环线性查找，大工作流应改 HashMap | 中 |
| 全局 SIDECAR 单例 | 浏览器录制进程全局共享，并发工作流可能冲突 | 低 |
