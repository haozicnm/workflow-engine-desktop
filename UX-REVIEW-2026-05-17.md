## 二、🔴 可访问性问题（Critical）

### 1. Settings 单选按钮组 — 无 ARIA 语义
**位置：** `Settings.vue` 浏览器节点选择、主题选择
**问题：** 使用 `<label>` + `<div>` 模拟 radio button，无 `role="radio"`、`aria-checked`、键盘导航
```html
<!-- 现状 -->
<label class="flex items-center gap-2 cursor-pointer" @click="browserMode = 'auto'">
  <div :class="['w-4 h-4 rounded-full border-2', browserMode === 'auto' ? 'border-primary' : 'border-border']">
    <div v-if="browserMode === 'auto'" class="w-2 h-2 rounded-full bg-primary m-0.5" />
  </div>
  <span>自动检测</span>
</label>

<!-- 建议 -->
<div role="radiogroup" aria-label="浏览器选择">
  <button role="radio" :aria-checked="browserMode === 'auto'"
          @click="browserMode = 'auto'"
          @keydown.left="moveToPrev" @keydown.right="moveToNext">
    自动检测
  </button>
  ...
</div>
```

### 2. Toast 通知 — 无 role="alert"
**位置：** `Toast.vue`
**问题：** 屏幕阅读器不会自动朗读 toast 消息
**修复：** 添加 `role="alert" aria-live="polite"`

### 3. Sidebar 导航 — 缺 ARIA 标注
**位置：** `Dashboard.vue` sidebar 菜单项
**问题：**
- `SidebarMenuButton` 无 `aria-current="page"` 标识当前项
- `SidebarGroupLabel` 是 `<span>` 非 heading，无导航区域语义
- Sidebar `<aside>` 无 `aria-label`

### 4. 多处 `<div @click>` 替代 `<button>`
**位置：**
- `RunHistory.vue` — 运行记录卡片展开/折叠
- `Editor.vue` — 日志面板展开/折叠
- `App.vue` — 面板关闭遮罩

**修复：** 全部改为 `<button>` 并添加 `aria-expanded` / `aria-label`

## 三、🟡 交互问题（High）

### 5. Toast 重叠 + 无法手动关闭
**位置：** `Toast.vue` + `useToast.ts`
**问题：** 多条 toast 在同一位置（`top-4 right-4`）堆叠；无手动关闭按钮；无 hover 暂停
**建议：** 参考 shadcn-vue sonner 组件模式，垂直堆叠 + 关闭按钮

### 6. `window.confirm()` 破坏视觉一致性
**位置：** `Editor.vue` 删除工作流
**问题：** 原生浏览器弹窗 vs 全站 styled 组件
**修复：** 使用 shadcn AlertDialog 或自定义 Dialog 组件

### 7. `useToast` 非单例 Bug
**位置：** `src/composables/useToast.ts`
**问题：** 每次调用创建独立 toasts 数组，外部组件调用会静默失败
**修复：** 将 toasts/nextId 提到模块作用域顶部：
```typescript
const toasts = ref<ToastItem[]>([])
let nextId = 0
export function useToast() { return { toasts, show, ... } }
```

### 8. ⋯ 菜单定位硬编码
**位置：** `Editor.vue` `toggleMenu`
**问题：** `left: ${rect.right - 176}px` 靠近右边缘时溢出
**建议：** 使用 `@floating-ui/vue` 或计算 `right: Math.min(0, viewportWidth - rect.right - 176)`

## 四、🟢 状态覆盖（Medium）

### 9. Dashboard 缺 loading 态
**位置：** `Dashboard.vue`
**问题：** `loading` ref 存在但未渲染任何 UI，用户看到空白列表
**修复：** 加骨架屏或 spinner

### 10. Dashboard 缺 empty 态
**位置：** `Dashboard.vue` 侧边栏列表
**问题：** 无工作流时列表完全空白，无引导文字
**建议：** 在欢迎页有引导，但侧边栏内也应显示 "暂无工作流，点击 + 新建"

### 11. 编辑器无"未保存更改"守卫
**位置：** `Editor.vue`
**问题：** 切换到其他工作流或关闭时不提示保存
**建议：** 切换前检查 `store.dirty`，弹出确认对话框

### 12. 运行中的记录不显示已用时间
**位置：** `RunHistory.vue`
**问题：** `finished_at` 为 null 时显示 `—`
**建议：** 显示 `已运行 ${elapsed}`,实时更新或在列表中标注"运行中"

## 五、🎨 视觉设计建议（基于 Objects in Space 系统）

### 当前状态
- GitHub Primer 配色 — 功能性强但偏"开发者工具"感
- `bg-card: #f6f8fa`（浅灰卡片）+ `border` + 细微阴影
- 系统字体栈：`system-ui, -apple-system, 'PingFang SC', 'Microsoft YaHei'`

### 推荐方案：A 极光幻影（AI/科技/工具）

```
Base    #EEEBFF  极浅紫 — 页面背景
Flow    #D6E5FA  极光蓝 — 辅助弥散色
Spark   #D1F5EA  浅青   — 点睛强调色
Anchor  #312C51  深紫   — 文字主色
```

**具体改进点：**

| 当前 | 建议 |
|------|------|
| 纯白/深色背景 `bg-background` | 添加 Auroral Glow 背景层（6椭圆弥散光效） |
| `box-shadow` 卡片 | 改为纯白卡片浮在 Glow 上，去阴影 |
| 圆角 `0.5rem`（8px） | 主卡片提升到 `24px`，次级 `14-16px` |
| system-ui 字体 | 中文用 HarmonyOS Sans SC，数字用 Space Grotesk |
| 字重混用 | 只用 700 Bold / 400 Regular，禁止斜体 |
| 导航栏 | 透明→滚动变色（backdrop-blur）+ 目录展开 |
| 按钮配色 | Primary 锚点色，Hover Flow 色 |

**快速落地建议（渐进式，不改架构）：**

1. **字体替换**（改动最小，效果最大）：
```html
<!-- 添加到 index.html -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/harmonyos-sans-sc@1.0.0/index.css">
<link href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;700&display=swap" rel="stylesheet">
```
```css
/* style.css */
body { font-family: 'HarmonyOS Sans SC', 'PingFang SC', 'Microsoft YaHei', sans-serif; }
.number, .data, .mono { font-family: 'Space Grotesk', monospace; }
```

2. **卡片圆角统一**：
```css
--radius-lg: 24px;  /* 主卡片 */
--radius-md: 14px;  /* 次级容器 */
```

3. **去阴影**（Objects in Space 核心理念）：
```css
.card { box-shadow: none; }
```

> ⚠️ 完整 Glow 系统需要重构背景层，建议分两个迭代：先落地字体+圆角+去阴影，再考虑 Glow。
